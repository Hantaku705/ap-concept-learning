# /should-skill - スキル・コマンド化判定

現在のセッションで行った作業を振り返り、スキルまたはコマンドとして保存すべきパターンがあるか判定する。

## 実行手順

1. **セッション振り返り**: 今回のセッションで行った作業を一覧化する
   - TodoWriteの履歴
   - `git diff --name-only` で変更ファイルを確認
   - 直近の会話から作業パターンを抽出

2. **5条件で判定**: 各パターンを以下の条件で評価し、該当数をカウントする

   | # | 条件 | 該当? |
   |---|------|-------|
   | 1 | 同じ手順を3回以上繰り返した | |
   | 2 | 複数プロジェクトで再利用できる | |
   | 3 | 手順が5ステップ以上ある | |
   | 4 | 判断基準（if/then）が含まれる | |
   | 5 | 忘れやすい・間違えやすい手順がある | |

   **3つ以上該当 → 提案対象**

3. **Skill vs Command 判断**: 該当パターンについて分類する
   - **Skill**（`_claude-code/skills/`）: 考え方・判断基準・チェックリスト・フレームワーク → **5条件中3つ以上で提案**
   - **Command**（`_claude-code/commands/`）: ツール実行・エージェント起動・ファイル操作の手順 → **5条件中3つ以上 + 以下を全て満たす場合のみ提案**
     - プロジェクトを問わず使える（特定ドメインに依存しない）
     - 毎セッションまたは毎週使える頻度がある
     - `/handoff`、`/resume`、`/commit-push-pr` レベルの普遍性がある

   **Commandは数が増えると覚えられないため、本当に普遍的なものだけ提案する。迷ったらSkillにする。**

4. **提案**: 以下のフォーマットでユーザーに提案する

   ```
   ## スキル/コマンド化提案

   ### パターン: [パターン名]
   - 種別: Skill / Command
   - 該当条件: [1,3,4] (3/5)
   - 保存先: `_claude-code/skills/xxx.md` or `_claude-code/commands/xxx.md`
   - 概要: [1行説明]

   作成しますか？
   ```

5. **作成または終了**:
   - ユーザー承諾 → 即座にファイル作成 + CLAUDE.md既存一覧に追記
   - 該当なし → 「今回のセッションではスキル/コマンド化の該当パターンはありませんでした。」と報告
