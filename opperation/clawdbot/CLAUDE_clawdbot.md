# Clawdbot 完全ガイド

## 概要

Clawdbotは、メッセージングアプリ内で動作するAIアシスタントです。ChatGPTやClaudeとは異なり、既存のアプリ（WhatsApp、Telegram、iMessage、Slack、Discordなど）内で動作し、会話の履歴を記憶し、プロアクティブにメッセージを送信できる点が特徴です。

---

## Clawdbotとは何か？

### 基本的な考え方

- **ChatGPT/Claude**: ウェブサイトにアクセスして質問し、回答をコピー&ペーストする
- **Clawdbot**: スマートフォン内で動作し、既存のメッセージングアプリで友達のようにテキストでやり取りできる

### 特徴

1. **統一された会話履歴**: スマホ、PC、タブレットで同じ会話が継続
2. **記憶機能**: 過去の会話、好み、以前に言及した内容をすべて記憶
3. **プロアクティブな通知**: AIから先にメッセージを送信可能
4. **実際のタスク実行**: 質問に答えるだけでなく、実際にコンピュータ上で作業を実行

---

## なぜ注目されているのか？

### 1. 実際に記憶する

- Siriは昨日の会話を覚えていない
- Clawdbotは過去の会話、好み、2週間前に言及した内容まで記憶
- 時間をかけてコンテキストを構築し、より良いサポートを提供

### 2. 先にメッセージを送ってくる

通常のAIはユーザーが開くのを待つが、Clawdbotは以下のような通知を送信可能：

- 「緊急メールが3件、20分後に会議があります」
- 「見ている株が5%下落しました」
- 「明日は天気が悪いので、予定を変更した方が良いかもしれません」

### 3. コンピュータ上で実際に作業を実行

- フォーム入力
- メール送信
- ファイル移動
- プログラム実行
- ブラウザ操作

実際の使用例：あるユーザーは、Netflixを見ながらベッドでClawdbotにテキストで指示を送り、ノートPCを開かずにウェブサイト全体を再構築した。

## 仕組み

### シンプルな説明

1. Clawdbotがコンピュータ（自分のPCまたは$5のクラウドサーバー）で動作
2. メッセージングアプリに接続
3. テキストでやり取り
4. そのコンピュータ上でタスクを実行可能

### 技術的な説明

- **Gateway**: バックグラウンドで動作する「交換台」のようなもの
- メッセージがWhatsApp、Telegramなどから入ってくる
- GatewayがAIにルーティング
- AIが思考し、応答し、アクション（ブラウザを開く、スクリプトを実行するなど）をトリガー可能
- すべてのデータは自分のマシン上に保存（Claude/ChatGPTへのAI呼び出しを除く）

## 実際の使用例

### 1. 朝のブリーフィング
起床時に重要なメール、その日のカレンダー、処理すべきタスクの要約をスマホに配信。

### 2. 健康トラッキング
「WHOOPに接続して、毎日の要約を送って」- 5分でセットアップ。自動的にフィットネスインサイトを取得。

### 3. メール管理
「これらのニュースレターをすべて配信停止して」- メールにログインし、不要なものを検索して処理。

### 4. リサーチアシスタント
「東京のホテル近くのベストレストラン5選を見つけて」- 検索、比較、オプション提示をすべてテキストスレッドで実行。

### 5. タスク自動化
「毎週金曜日の午後5時に、今週達成したことの要約を送って」- 一度設定すれば、永遠に実行される。

### 6. 高度な使用例
あるユーザーは、Clawdbotにカスタム瞑想を書かせ、AI音声で音声を生成し、環境音楽を追加して、毎朝自動配信するシステムを構築。

## インストール手順

### 【ステップ1】Clawdbotをインストール

**npmの場合：**
```bash
npm install -g clawdbot@latest
```

**pnpmの場合：**
```bash
pnpm add -g clawdbot@latest
```

**または公式のワンライナー：**
```bash
curl -fsSL https://clawd.bot/install.sh | bash
```

### 【ステップ2】オンボーディングウィザードを実行

```bash
clawdbot onboard --install-daemon
```

ウィザードが起動し、以下を対話的に設定：
- Gateway
- ワークスペース
- チャンネル
- スキル

### 【ステップ3】LLMを用意する

#### A. LocalLLM（プライバシー重視）

- Ollama
- llama.cpp
- LM Studio

**メリット**: 会話内容がローカルで完結し、外部クラウドに送信されない

#### B. 外部API（性能重視）

- **Claude API**（公式推奨はClaude Opus 4.5）
- OpenAI API

**推奨**: 長いコンテキストやプロンプトインジェクション耐性の観点から、Anthropic Claude Opus 4.5が推奨されています。

### 【ステップ4】チャットUIと接続

利用したいチャットツールのBotトークンを設定：
- Telegram
- Slack
- Discord
- WhatsApp
- iMessage
- その他

### 【ステップ5】Gatewayを起動

```bash
clawdbot gateway
```

この時点で、AIアシスタントとして利用可能になります。
