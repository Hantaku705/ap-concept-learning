# /create-skill - 会話からコマンドスキルを生成

現在の会話のやり取りを分析し、再利用可能なコマンドスキル（.md）を自動生成します。

## 実行手順

### 1. 会話コンテキストの分析

現在のセッションの会話履歴を分析し、以下を特定:

- **目的**: ユーザーが達成しようとしたこと
- **入力**: 最初に与えられた情報（ファイル、URL、指示等）
- **出力**: 最終的に生成されたもの（ファイル、デプロイ等）
- **ステップ**: 目的達成のために行った一連の作業

### 2. ワークフローパターンの抽出

会話から繰り返し可能なワークフローを抽出:

```
入力 → ステップ1 → ステップ2 → ... → 出力
```

**抽出観点**:
- 汎用化できるステップ（具体的なファイル名を変数化）
- 必須ステップ vs オプショナルステップ
- 分岐点（条件によって変わる処理）

### 3. スキル名・説明の提案

ユーザーに以下を提案:

- **スキル名**: `[動詞]-[名詞]` 形式（例: `analyze-pdf`, `create-webapp`）
- **説明**: 10〜30文字の1行説明
- **保存先**: グローバル（`~/.claude/commands/`）or プロジェクト（`.claude/commands/`）

### 4. スキルファイルの生成

以下のテンプレートでスキルファイルを生成:

```markdown
---
description: "/[スキル名] - [1行説明]"
---

# /[スキル名] - [説明]

[概要説明（1-2文）]

## 前提条件

- [必要な入力・環境]

## 実行手順

1. [ステップ1]
2. [ステップ2]
...

## 出力

- [生成されるファイル・結果]

## 使用例

```
/[スキル名]
```
```

### 5. ファイル保存

1. 保存先を確認（グローバル or プロジェクト）
2. ファイルを保存: `[保存先]/[スキル名].md`
3. 保存完了を報告

### 6. 検証

- `/[スキル名]` で呼び出せることを確認（ユーザーに案内）
- 必要に応じてスキル内容を調整

---

## オプション

引数でカスタマイズ可能:

| 引数 | 説明 | 例 |
|------|------|-----|
| `--name [名前]` | スキル名を指定 | `/create-skill --name analyze-pdf` |
| `--global` | グローバル保存（デフォルト） | `/create-skill --global` |
| `--project` | プロジェクト保存 | `/create-skill --project` |
| `--dry-run` | 保存せずプレビューのみ | `/create-skill --dry-run` |

---

## 注意事項

- 会話にやり取りがない場合はスキル生成できない旨を案内
- 既存スキルと同名の場合は上書き確認
- 機密情報（APIキー等）はスキルに含めない
