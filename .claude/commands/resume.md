# /resume - セッション再開時の読み込み

前回のセッションから作業を再開します。

## 実行手順

1. HANDOFF.md を読み込む
2. 内容をユーザーに報告:
   - 前回の完了/作業中タスク
   - 次のアクション（優先順位付き）
   - 未解決の問題
   - 未コミット変更の有無
3. git status で現在の状態を確認
4. 「何から始めますか？」と確認

## 出力フォーマット

```
## 前回の状態

### 完了済み
- [タスク一覧]

### 作業中
- [タスク一覧]

### 次のアクション（優先順）
1. [アクション]
2. [アクション]

### 未解決の問題
- [問題]

---

**今回は何から始めますか？**
```

## 大容量ファイル対応

### トークン超過エラー時
HANDOFF.md読み込みで「トークン超過」エラーが発生した場合:

1. エラーを検知
2. ユーザーに案内:
   ```
   HANDOFF.mdが大きすぎます（25,000トークン上限超過）。
   アーカイブ処理が必要です。
   → /handoff を実行してアーカイブ処理を行ってください。
   ```
3. 代替として先頭200行を読み込んで概要を報告

### アーカイブ参照
- HANDOFF.md内に「HANDOFF_ARCHIVE.mdを参照」リンクがある場合は案内
- 過去のセッション詳細が必要な場合、必要に応じてHANDOFF_ARCHIVE.mdも読み込み

---

## 注意事項

- HANDOFF.md が存在しない場合は新規セッションとして案内
- 未コミット変更がある場合は先に確認を促す
- git stash がある場合は stash pop を提案
- **トークン超過エラー時は /handoff でアーカイブを促す**
