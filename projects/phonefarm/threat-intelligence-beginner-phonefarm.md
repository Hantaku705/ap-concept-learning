## 概要

### Phone Farmとは

複数のスマートフォンを同時に管理・操作し、ソーシャルメディアでの不正なエンゲージメント（フォロワー水増し、いいね操作、再生数操作）を行うシステム。

### 初心者業者の特徴

| 特徴 | 詳細 |
|------|------|
| 規模 | 5-20台程度 |
| 投資額 | 5-15万円 |
| 技術レベル | 基本的なスクリプト、GUI自動化ツール |
| 検出回避 | 不十分（パターンが単純） |

### 不正行為の種類

- フォロワー販売（1000フォロワー = ¥500-2000）
- いいね販売（1000いいね = ¥300-1000）
- 再生数操作（10000再生 = ¥500-1500）
- コメント投稿代行

|------|-------------------|---------------|
| タッチイベント | ADB/アクセシビリティ経由 | **本物のタッチと同一** |

#### 検出クエリ例（物理クリッカー）

```sql
-- タップ間隔の規則性検出
SELECT
    user_id,
    STDDEV(interval_ms) as interval_stddev,
    AVG(interval_ms) as interval_avg,
    COUNT(*) as tap_count
FROM (
    SELECT
        user_id,
        EXTRACT(EPOCH FROM (timestamp - LAG(timestamp) OVER (
            PARTITION BY user_id ORDER BY timestamp
        ))) * 1000 as interval_ms
    FROM tap_events
    WHERE timestamp > NOW() - INTERVAL 1 HOUR
) intervals
WHERE interval_ms IS NOT NULL
GROUP BY user_id
HAVING
    interval_stddev < 50  -- 標準偏差50ms未満は機械的
    AND tap_count > 100;

-- ジャイロセンサー静止検出
SELECT
    user_id,
    AVG(ABS(gyro_x)) + AVG(ABS(gyro_y)) + AVG(ABS(gyro_z)) as total_movement
FROM sensor_data
WHERE timestamp > NOW() - INTERVAL 1 HOUR
GROUP BY user_id
HAVING total_movement < 0.01;  -- ほぼ完全静止
```

## 4. 自動スクロール/エンゲージメント実装

### 4.1 ADBによる実装（最も基本）

```bash
#!/bin/bash
# tiktok_auto_scroll.sh - 初心者が使う基本スクリプト

DEVICES=$(adb devices | grep device$ | cut -f1)

while true; do
    for device in $DEVICES; do
        # 上にスワイプ（次の動画へ）
        adb -s $device shell input swipe 540 1500 540 500 300 &
    done

    # 待機（3-8秒のランダム）
    sleep $((RANDOM % 6 + 3))

    # 30%の確率でいいね
    if [ $((RANDOM % 10)) -lt 3 ]; then
        for device in $DEVICES; do
            adb -s $device shell input tap 950 1200 &
        done
    fi
done
```

**検出ポイント**:
- `input swipe` のパラメータが固定（540 1500 540 500 300）
- スワイプ軌跡が完全な直線
- 全端末が同一タイミングで動作

```javascript
    minViewTime: 3000,      // 最小視聴時間(ms)

        // 動画視聴

            likeVideo();
        }

        // フォロー（確率で実行）

            followUser();
        }

        // 次の動画へスクロール

    }
}

// スワイプで次の動画へ
function scrollToNext() {
    var startX = device.width / 2;
    var startY = device.height * 0.7;
    var endY = device.height * 0.3;
    var duration = random(300, 500);

    swipe(startX, startY, startX, endY, duration);
    sleep(random(500, 1000));
}

// いいね
function likeVideo() {
    // 画面右側のハートアイコンをタップ
    var x = device.width * 0.9;
    var y = device.height * 0.5;

// フォロー
function followUser() {
    // プロフィールアイコンをタップ

    // 戻る

    sleep(500);
}

// 実行
main();
```

### 4.4 Python + ADBによる実装（やや上級初心者）

```python
#!/usr/bin/env python3
# tiktok_farm.py - Python制御スクリプト

import subprocess
import time
import random
import threading

class TikTokFarm:
    def __init__(self):
        self.devices = self.get_devices()
        self.running = True

    def get_devices(self):
        """接続デバイス一覧を取得"""
        result = subprocess.run(
            ["adb", "devices"],
            capture_output=True,
            text=True
        )
        lines = result.stdout.strip().split('\n')[1:]
        devices = [line.split('\t')[0] for line in lines if 'device' in line]
        return devices

    def swipe_up(self, device_id):
        """上にスワイプ"""
        # 座標に微小なランダム性を追加（初心者の改良版）
        start_x = 540 + random.randint(-20, 20)
        start_y = 1500 + random.randint(-30, 30)
        end_x = 540 + random.randint(-20, 20)
        end_y = 500 + random.randint(-30, 30)
        duration = random.randint(250, 400)

        cmd = f"adb -s {device_id} shell input swipe {start_x} {start_y} {end_x} {end_y} {duration}"
        subprocess.run(cmd, shell=True)

    def tap_like(self, device_id):
        """いいねをタップ"""
        x = 950 + random.randint(-10, 10)
        y = 1200 + random.randint(-10, 10)
        cmd = f"adb -s {device_id} shell input tap {x} {y}"
        subprocess.run(cmd, shell=True)

    def run_device(self, device_id):
        """1台のデバイスを自動操作"""
        while self.running:
            # 視聴時間
            view_time = random.uniform(3, 8)
            time.sleep(view_time)

            # スワイプ
            self.swipe_up(device_id)
            time.sleep(random.uniform(0.5, 1))

            # 30%でいいね
            if random.random() < 0.3:
                self.tap_like(device_id)

    def start(self):
        """全デバイスで開始"""
        threads = []
        for device in self.devices:
            t = threading.Thread(target=self.run_device, args=(device,))
            t.start()
            threads.append(t)
            time.sleep(0.5)  # 開始タイミングをずらす

        try:
            while True:
                time.sleep(1)
        except KeyboardInterrupt:
            self.running = False
            for t in threads:
                t.join()

if __name__ == "__main__":
    farm = TikTokFarm()
    print(f"検出デバイス: {farm.devices}")
    farm.start()
```

**検出ポイント**:
- ランダム性を追加しても、分布が均一（人間は正規分布的）
- スレッド開始の0.5秒間隔が規則的
- 全デバイスの行動パターンが類似

### 5.5 検出クエリ例

```sql
-- 同一機種からの異常アクセス
SELECT
    device_model,
    COUNT(DISTINCT user_id) as user_count,
    COUNT(*) as action_count
FROM user_actions
WHERE timestamp > NOW() - INTERVAL 1 HOUR
GROUP BY device_model
HAVING user_count > 5
ORDER BY user_count DESC;

-- 同一IPからの複数アカウント
SELECT
    ip_address,
    COUNT(DISTINCT user_id) as account_count,
    ARRAY_AGG(DISTINCT user_id) as user_ids
FROM login_logs
WHERE timestamp > NOW() - INTERVAL 24 HOUR
GROUP BY ip_address
HAVING account_count > 2;

-- 操作タイミングの同期検出
SELECT
    DATE_TRUNC('second', timestamp) as action_second,
    COUNT(*) as simultaneous_actions,
    ARRAY_AGG(user_id) as users
FROM user_actions
WHERE action_type = 'like'
  AND timestamp > NOW() - INTERVAL 1 HOUR
GROUP BY action_second
HAVING simultaneous_actions > 3;

-- 視聴時間分布の異常検出
SELECT
    user_id,
    AVG(view_duration) as avg_duration,
    STDDEV(view_duration) as stddev_duration,
    COUNT(*) as view_count
FROM video_views
WHERE timestamp > NOW() - INTERVAL 24 HOUR
GROUP BY user_id
HAVING
    stddev_duration < 1.0  -- 標準偏差が小さすぎる
    AND view_count > 100;
```
